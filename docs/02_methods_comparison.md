# Сравнение подходов к улучшению изображений

## Обзор подходов

В этом разделе мы проведём сравнительный анализ различных подходов к улучшению изображений с учётом требований задачи:
- ✅ Качество улучшения
- ⚡ Скорость работы (real-time)
- 💾 Потребление ресурсов
- 🎯 Сохранение идентичности

---

## 1. Классические методы (Traditional Methods)

### 1.1 Гамма-коррекция

**Принцип**: Нелинейное преобразование яркости $I_{out} = I_{in}^\\gamma$

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | Высокая скорость, детерминированность, простота реализации |
| **Недостатки** | Ограниченная эффективность, не учитывает контекст изображения |
| **Применение** | Базовая коррекция экспозиции |
| **Скорость** | ~1 мс |
| **Качество** | 3/10 |

### 1.2 Гистограммная эквализация (Histogram Equalization)

**Принцип**: Перераспределение яркости для равномерного покрытия диапазона

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | Автоматическая адаптация к изображению, высокая скорость |
| **Недостатки** | Может создавать неестественные эффекты, не учитывает семантику |
| **Применение** | Улучшение контраста |
| **Скорость** | ~5 мс |
| **Качество** | 4/10 |

### 1.3 CLAHE (Contrast Limited Adaptive Histogram Equalization)

**Принцип**: Адаптивная гистограммная эквализация с ограничением контраста

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | Лучше сохраняет детали, меньшие артефакты |
| **Недостатки** | Все ещё ограниченная эффективность, требует тюнинга параметров |
| **Применение** | Улучшение локального контраста |
| **Скорость** | ~10 мс |
| **Качество** | 5/10 |

## 2. CNN-based методы

### 2.1 Полносверточные сети (FCN)

**Принцип**: Энд-то-энд обучение для отображения input → output

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | Высокое качество, возможность обучения на данных |
| **Недостатки** | Вычислительная сложность, большой размер модели |
| **Применение** | Общее улучшение изображений |
| **Скорость** | ~100 мс |
| **Качество** | 7/10 |
| **Размер модели** | ~50 MB |

### 2.2 GAN-подходы (Generative Adversarial Networks)

**Принцип**: Генератор улучшает изображение, дискриминатор отличает от референса

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | Высокое визуальное качество, реалистичные результаты |
| **Недостатки** | Нестабильность обучения, вычислительная сложность |
| **Применение** | Художественная обработка, стилизация |
| **Скорость** | ~500 мс |
| **Качество** | 8/10 |
| **Размер модели** | ~200 MB |

### 2.3 Диффузионные модели (Diffusion Models)

**Принцип**: Постепенное улучшение через обратный диффузионный процесс

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | State-of-the-art качество, высокая гибкость |
| **Недостатки** | Очень медленные, требовательны к ресурсам |
| **Применение** | Высококачественная обработка, художественные эффекты |
| **Скорость** | ~5000 мс |
| **Качество** | 10/10 |
| **Размер модели** | ~1000 MB |

## 3. LUT-based методы (РЕКОМЕНДОВАНЫ)

### 3.1 Статический 3D LUT (Look-Up Table)

**Принцип**: Предвычисленная таблица цветокоррекции для всех комбинаций RGB

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | Экстремально высокая скорость, компактность, детерминированность |
| **Недостатки** | Ограниченная гибкость, требует обучения/генерации таблиц |
| **Применение** | Цветокоррекция, фильtrl-эффекты |
| **Скорость** | ~1 мс ⚡⚡⚡ |
| **Качество** | 5/10 |
| **Размер модели** | ~2 MB |

### 3.2 Адаптивный LUT (Parametric LUT) [ВЫБОР]

**Принцип**: LUT с параметрами, адаптирующимися под изображение

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | Баланс скорости и гибкости, хорошая обобщающая способность |
| **Недостатки** | Сложнее реализации, чем статические LUT |
| **Применение** | Умная цветокоррекция |
| **Скорость** | ~5-20 мс ⚡⚡ |
| **Качество** | 7/10 ⭐ |
| **Размер модели** | ~15 MB |

### 3.3 Гибридные CNN+LUT подходы

**Принцип**: CNN извлекает параметры, LUT применяет цветокоррекцию

| Характеристика | Значение |
|----------------|----------|
| **Преимущества** | Высокое качество при низкой вычислительной сложности |
| **Недостатки** | Более сложная архитектура |
| **Применение** | Real-time улучшение с высоким качеством |
| **Скорость** | ~20-50 мс ⚡ |
| **Качество** | 8/10 ⭐⭐ |
| **Размер модели** | ~30 MB |
